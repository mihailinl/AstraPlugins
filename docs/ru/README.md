# Разработка плагинов Astra

Создавайте плагины для [Astra](https://github.com/Stella) — цифрового ассистента на базе ИИ — на Rust, Python или TypeScript.

Плагины — это независимые процессы, которые Astra запускает в качестве сайдкаров. Они взаимодействуют с демоном через gRPC и могут предоставлять ИИ-инструменты, обеспечивать бэкенды TTS/STT, добавлять пользовательские действия и триггеры в Граф команд, внедрять UI-панели или работать как полноценные клиенты демона.

## Архитектура вкратце

```
┌────────────────────────┐     gRPC     ┌────────────────────────┐
│        Astra           │◀────────────▶│        Plugin          │
│        daemon          │   localhost  │      (sidecar)         │
│                        │              │                        │
│ PluginHostService ─────┼────────────▶│ HostClient             │
│                        │              │                        │
│ plugin-capability ◀────┼──────────────│ PluginCapability       │
│   service client       │              │   service              │
└────────────────────────┘              └────────────────────────┘
```

- Демон запускает каждый плагин как отдельный процесс, передавая `--daemon-addr`, `--plugin-id` и, при необходимости, `--auth-token` в командной строке.
- Плагин поднимает gRPC-сервер на случайном локальном порту, подключается обратно к `PluginHostService` демона и **регистрируется**, сообщая, какие возможности он реализует.
- После регистрации демон обращается к `PluginCapabilityService` плагина для вызовов инструментов, выполнения действий, TTS и событий жизненного цикла.
- Плагин использует `HostClient` для логирования, запуска триггеров, чтения собственной конфигурации, установки переменных или отправки событий в свои UI-iframe'ы. Плагины с возможностью клиента дополнительно получают полноценный `DaemonClient` с доступом к сервисам Chat, Voice, Command, Media, Monitor и Config.

## Выбор SDK

| Фактор | Rust | Python | TypeScript |
| --- | --- | --- | --- |
| Задержка запуска | ~10 мс (нативный бинарник) | ~300 мс (интерпретатор + импорт grpcio) | ~100 мс (холодный старт Node) |
| Потребление памяти | Минимальное | Максимальное | Среднее |
| Размер бандла | ~5–10 МБ бинарник | ~100 КБ исходников + venv, управляемый демоном | ~200 КБ бандл (esbuild) |
| Лучше всего для | Критичной к производительности интеграции с системой, провайдеров TTS/STT | ИИ-инструментов, обработки данных, ML-библиотек | Web API, работы с JSON, интеграций с UI |
| Типобезопасность | Полная (во время компиляции) | Опциональная через подсказки (генерация схемы во время выполнения) | Полная (во время компиляции) |

Все три SDK являются первоклассными — каждая возможность доступна в каждом SDK. Выбирайте тот, который соответствует экосистеме библиотек, из которой вы хотите тянуть зависимости.
