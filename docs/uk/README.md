# Розробка плагінів Astra

Створюйте плагіни для [Astra](https://github.com/Stella) — цифрового асистента на основі штучного інтелекту — мовами Rust, Python або TypeScript.

Плагіни — це незалежні процеси, які Astra запускає як сайдкари. Вони спілкуються з демоном через gRPC і можуть надавати інструменти AI, забезпечувати бекенди TTS/STT, додавати власні дії та тригери до Command Graph, впроваджувати панелі UI або виступати як повноцінні клієнти демона.

## Архітектура коротко

```
┌────────────────────────┐     gRPC     ┌────────────────────────┐
│        Astra           │◀────────────▶│        Plugin          │
│        daemon          │   localhost  │      (sidecar)         │
│                        │              │                        │
│ PluginHostService ─────┼────────────▶│ HostClient             │
│                        │              │                        │
│ plugin-capability ◀────┼──────────────│ PluginCapability       │
│   service client       │              │   service              │
└────────────────────────┘              └────────────────────────┘
```

- Демон запускає кожен плагін як окремий процес, передаючи у командному рядку `--daemon-addr`, `--plugin-id` та, за потреби, `--auth-token`.
- Плагін запускає gRPC-сервер на випадковому локальному порту, підключається назад до `PluginHostService` демона та **реєструється** — повідомляє, які можливості він реалізує.
- Після реєстрації демон викликає `PluginCapabilityService` плагіна для викликів інструментів, виконання дій, TTS та подій життєвого циклу.
- Плагін використовує `HostClient`, щоб журналювати, активувати тригери, читати власну конфігурацію, встановлювати змінні або надсилати події у свої iframe-и UI. Плагіни з можливістю client-режиму додатково отримують повний `DaemonClient` з доступом до служб Chat, Voice, Command, Media, Monitor і Config.

## Як обрати SDK

| Фактор | Rust | Python | TypeScript |
| --- | --- | --- | --- |
| Затримка запуску | ~10 мс (нативний бінарник) | ~300 мс (інтерпретатор + імпорт grpcio) | ~100 мс (холодний старт Node) |
| Споживання пам'яті | Найменше | Найбільше | Середнє |
| Розмір бандла | бінарник ~5–10 МБ | ~100 КБ вихідного коду + venv під керуванням демона | бандл ~200 КБ (esbuild) |
| Найкраще для | Продуктивність, системна інтеграція, постачальники TTS/STT | Інструменти AI, обробка даних, бібліотеки ML | Веб-API, робота з JSON, інтеграції UI |
| Типобезпека | Повна (під час компіляції) | Опціональна через підказки (генерація схеми на льоту) | Повна (під час компіляції) |

Усі три SDK є першокласними — кожна можливість доступна у кожному SDK. Обирайте той, що відповідає екосистемі, з якої ви хочете підтягувати бібліотеки.
